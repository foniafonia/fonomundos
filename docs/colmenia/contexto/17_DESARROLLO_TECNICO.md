# Desarrollo técnico

## Estado mental del proyecto

José no es programador tradicional, pero está aprendiendo a dirigir desarrollo con IA. La clave no es que programe todo, sino que sepa pedir, revisar, ordenar y decidir.

## Herramientas

- Codex.
- Claude Code.
- GitHub.
- Vercel.
- Supabase.
- React / Vite / TypeScript / Tailwind en algunos proyectos.
- Posibles backends propios para clínica y comunidad.

## Problemas detectados

- Saturación de procesos locales.
- Next.js colgado por sobrecarga de CPU.
- Procesos duplicados.
- Confusión entre prototipo y producto.
- Falta de documentación centralizada.
- Riesgo de perder repositorios o versiones.

## Reglas técnicas

- Documentar cada repo.
- Mantener backups.
- No mezclar datos clínicos sensibles con prototipos.
- Modularizar.
- Evitar monolitos.
- Priorizar MVPs.
- No construir infraestructura imposible de mantener.

## Sistema clínico propio

El sistema para sustituir ClinicCloud debe construirse por fases:

1. Fichas, pacientes, agenda, pagos manuales.
2. Informes, presupuestos, plantillas.
3. Recordatorios y WhatsApp.
4. IA asistida para redacción y mejora.
5. Automatizaciones más avanzadas.
