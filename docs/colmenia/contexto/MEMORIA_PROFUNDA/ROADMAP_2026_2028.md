# ROADMAP 2026–2028 — Memoria profunda
*v1.0 · Consolida docs 21, 42, 180–181, 323. Hipótesis de evolución, no compromiso cerrado.*

## Resumen ejecutivo
Prioridad por orden: **comunidad → herramientas → IA integrada → plataforma propia → ecosistema conectado.** Primero la gente y la utilidad diaria; la plataforma, después. Visión: Logoped-IA como sistema operativo del profesional de logopedia/educación/salud.

## Prioridades tecnológicas (orden)
1. Comunidad (Telegram/COLMENIA activos).
2. Herramientas útiles (informes, materiales, prototipos que pasen el test del lunes).
3. IA integrada (asistentes/GPTs validados, p. ej. Logoped-IA Shield).
4. Plataforma propia (COLMENIA como sistema, y sustituto modular de ClinicCloud: agenda, historia clínica, informes, documentos, IA).
5. Ecosistema conectado (todo comunicado; analítica; perfiles).

## FonoMundos (línea de innovación)
Mundo persistente · analítica clínica · perfil paciente/profesional · editor de actividades · cooperativo · IA adaptativa · integración con COLMENIA. [HECHO, docs 42/323]

## Visión 2035
Un profesional acompañado por un ecosistema inteligente: una puerta en vez de veinte apps; la información aparece cuando se necesita; informes que se revisan (no se escriben desde cero); sesiones que se adaptan; comunidad internacional. La tecnología desaparece del primer plano; vuelve el paciente. [HECHO, doc LOGOPEDIA_OS]

## Riesgos
Querer construir la plataforma antes que la utilidad · dependencia de desarrolladores · sostenibilidad económica (objetivo 2–3k€/mes con ~2h/día debe guiar prioridades: producto con retorno antes que proyectos infinitos).
