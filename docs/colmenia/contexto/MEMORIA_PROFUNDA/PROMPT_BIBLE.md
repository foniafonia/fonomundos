# PROMPT BIBLE — Memoria profunda
*v1.0 · Centraliza los prompts estratégicos reutilizables. Cada prompt: rol, contexto, objetivo, entradas, restricciones, salida, calidad. Consolida docs 23/240/13 + línea editorial.*

## Principio
Un prompt sin criterio es una receta vacía. Estos prompts asumen SIEMPRE el ADN del proyecto (ver `/CLAUDE.md`): problema antes que tecnología, no regalar el núcleo, IA no sustituye, alcance multidisciplinar, tono humano-clínico sin humo.

## 1. Prompt · Newsletter COLMENIA (Cuaderno de laboratorio)
- **Rol:** editor de un cuaderno de laboratorio profesional.
- **Objetivo:** una edición que gire alrededor de UNA pregunta potente, descubierta por el lector (no anunciada en el título ni el primer párrafo).
- **Entradas (de José):** un hecho real de la semana (frase de un tester, error de prototipo, conversación, caso anonimizado) + qué forma de pensar le cambió.
- **Estructura narrativa:** hecho/escena → incomodidad/duda → la pregunta nace → pruebas (testers, vídeo, captura) al servicio de la pregunta → no cerrar, ampliar.
- **Restricciones:** no recopilación de enlaces; foto real (no banco de imágenes); una idea por edición; admitir al menos una cosa que aún no se sabe; anonimizar datos personales.
- **Salida:** texto en la voz de José (frases cortas, honesto, sin humo) + CTA discreto a Telegram.
- **Calidad:** ¿el lector termina pensando 5 min después? ¿demuestra criterio sin regalar el núcleo?

## 2. Prompt · Post de LinkedIn
- **Rol:** cabeza clínica que posiciona.
- **Objetivo:** autoridad + fricción intelectual que derive a Telegram/COLMENIA.
- **Restricciones:** no tutoriales completos; máx ~2/semana [capa viva, revisar]; uno de los 4 pilares (cuaderno abierto / criterio > herramienta / útil ya / humano-comunidad).
- **Filtro previo:** ¿a quién atrae? ¿qué autoridad construye? ¿qué conversación abre? ¿qué NO se regala? ¿cómo deriva? ¿profundidad adecuada al canal?

## 3. Prompt · Trabajar como Claude en este proyecto
Investigación profunda, escritura larga, documentación, análisis crítico. **Cuestionar, no confirmar.** Pocas opciones con jerarquía. Si dice "dámelo", entregar algo usable. Pasar toda propuesta por el test del lunes y el filtro del manual operativo.

## 4. Prompt · Definición de producto
problema · usuario principal · situación de uso · valor diferencial · MVP · validación · precio · canal de captación · conexión con COLMENIA · riesgo principal.
