# Woorkia, DAX, Mighty y plataformas

## Woorkia

Se trabajó una posible relación con Woorkia para desarrollo o acompañamiento técnico. Se definieron criterios importantes:

- José mantiene propiedad de contenidos educativos, clínicos, metodológicos y terapéuticos.
- La parte técnica puede pertenecer al desarrollador según contrato.
- Debe haber prioridad de encargo, pero no dependencia absoluta.
- Si Woorkia no cumple tiempos o calidad, se puede abrir a terceros.
- Deben evitarse cesiones ambiguas de FonoMundos o prototipos.

## DAX

DAX apareció como posible vía o interlocutor técnico/estratégico, pero también hubo cancelaciones o cambios. No debe asumirse que DAX es la base definitiva del proyecto.

## Mighty Networks

Mighty fue estudiado como plataforma para COLMENIA. Ventajas:

- comunidad,
- pagos,
- membresía,
- contenidos,
- estructura rápida,
- menos desarrollo propio.

Riesgos:

- apariencia genérica,
- sensación de Moodle avanzado,
- poca diferenciación,
- límites visuales,
- dependencia de plataforma.

## Decisión abierta

La decisión no es “Mighty sí/no”. La decisión real es arquitectura:

- ¿Landing propia + comunidad externa?
- ¿Mighty como núcleo visible?
- ¿Mighty como backend comunitario?
- ¿Sistema propio progresivo?

Debe resolverse con criterio de velocidad, diseño, coste, control y conversión.
